# 金蝶产品文档类型与模板

## 帮助文档体系结构框架

### 必选目录

| 一级目录名称 | 说明 |
|-------------|------|
| 整体介绍 | 帮助用户全面了解模块的核心业务场景、核心功能及价值、功能结构和业务流程等 |
| 功能介绍 | 帮助用户了解该功能的作用、涵盖的应用场景、关键字段的解释及使用注意事项等 |
| 场景指引 | 帮助用户了解特定场景的操作步骤和注意事项等（新增） |
| 常见问题 | 对用户在使用具体的功能或进行具体的操作过程中所遇到的问题进行分析解答 |

### 可选目录

| 一级目录名称 | 说明 |
|-------------|------|
| 重要通知 | 帮助用户了解产品重要公告内容 |
| 新特性公告 | 帮助用户详细了解最新的功能特性 |
| 升级指南 | 提供详细的升级步骤、注意事项和解决方案 |
| 安装部署指南 | 帮助用户了解软件或系统的安装与配置流程及操作 |
| 快速入门 | 帮助用户快速了解产品核心功能并快速上手使用 |
| 实践案例 | 基于典型客户形成案例，供同业客户参考 |
| 定制化开发手册 | 为定制化开发者提供开发指导 |
| API | 向开发者提供全面且清晰的接口说明 |
| SDK | 向定制开发者清晰、全面地介绍SDK的功能、场景、特点、使用方法等 |
| 系统管理员手册 | 系统管理员操作指南 |
| 购买指南 | 为客户提供全面、详尽的产品信息 |

### 目录结构建议

- 父子目录以总到分形式展开
- 并列子目录按业务流程构建
- 文档目录的命名须严格遵循规范，禁止随意更改（如将"实践案例"变更为"最佳实践"等不合规行为）

## 各文档类型详细模板

### 1. 整体介绍

**写作目标**：帮助用户全面了解模块的核心业务场景、核心功能及价值、功能结构和业务流程等

**标题建议**：XX模块整体介绍（例：移动应用整体介绍）

**内容结构**：

```
1 产品概述
  1.1 产品介绍（主要介绍产品是什么，可以解决什么问题，实现什么效果）
  1.2 产品结构图

2 业务流程
  2.1 流程介绍（概括性地说明业务流程，包括主流程内容及上下游关联说明）
  2.2 本应用与其他应用的关联（介绍本应用与相关应用之间的联系）

3 应用场景（介绍在什么情况下，可以使用什么样的功能完成什么业务，可链接到具体的"场景指引"的内容）

4 亮点特性（概述突出的特性，可链接到具体的"功能介绍"的内容）

5 名词解释（可选）

6 注意事项（可选，说明本应用在使用过程中可能遇到的问题及如何解决、需重点关注的事项等）

7 相关文档（可选）
```

---

### 2. 功能介绍

**写作目标**：帮助用户了解该功能的作用、涵盖的应用场景、关键字段的解释及使用注意事项等

**标题建议**：XX模块+具体功能+功能介绍

**内容结构**：

```
1 功能介绍（整体介绍功能是什么，可以解决什么问题，实现什么效果）

2 应用场景（介绍在什么情况下，可以使用什么样的功能完成什么业务。可直接外链"场景指引"文档）

3 系统路径

4 关键字段/按钮说明
  4.1 关键字段说明（仅核心字段或复杂的、需要特别说明的字段才需要在此说明其字段作用、使用方法、计算逻辑、使用限制条件、启用影响等）
  4.2 关键按钮说明（无特别含义的如"新增"、"删除"等通用操作按钮不必展示）

5 注意事项（可选，说明本功能在使用过程中可能遇到的问题及如何解决、需重点关注的事项等）

6 相关文档（可选）
```

**重要说明**：
- 功能介绍仅阐述该功能的作用、涵盖的应用场景、关键字段的解释及使用注意事项等，不描写操作步骤
- 关键字段指产品界面的核心字段或复杂的、需要特别说明的字段/按钮，强调其作用、使用方法、计算逻辑、启用影响、使用限制条件等
- 无特别含义的如"新增"、"删除"、"启用"、"禁用"、"提交"、"关闭"等通用操作按钮，若无特殊含义，则无需解释

---

### 3. 场景指引

**写作目标**：帮助用户了解特定场景的操作步骤和注意事项等

**标题建议**：XX特定场景下+XX产品/XX特性+操作指引（例：年中上预算系统时，已发生的实际数补录的场景指引）

**内容结构**：

```
1 业务场景（介绍在什么实际场景中，要完成什么特定任务）

2 解决方案
  2.1 解决方案总述（可选，介绍针对特定场景的核心解决方案及其预期效果）
  2.2 关键操作（注意：建议以场景化的角度来描述。通用的增、删、改、查如没有需要特别告知用户的限制条件或注意事项等，可以简要描述）
    2.2.1 前提条件
    2.2.2 操作步骤
    2.2.3 后续操作（可选）

3 注意事项（可选，强调实施过程中的关键细节、可能遇到的问题及其解决方案等）

4 相关文档（可选）
```

**重要说明**：
- 场景指引描写特定场景的操作步骤和注意事项等，需任务导向，内容应专注于指导用户在具体的实际场景中完成特定任务
- 操作步骤前后需要写"前提条件"和"后续操作"

---

### 4. 常见问题（问题分析类）

**写作目标**：对用户在使用具体的功能或进行具体的操作过程中所遇到的问题进行分析解答，帮助用户查找问题的原因，并提供可行的解决方法

**标题建议**：操作位置/界面+做什么+问题（例：智能会计平台应收款对账不平？/科目余额表本期合计与明细分类账对不上？）

**内容结构**：

```
1 问题描述（详细描述问题出现的业务场景和背景）

2 原因分析（可选）

3 解决方法（对于复杂且存在多种原因的问题，需详述问题出现的原因与对应解决的方法；对于简单问题，可直接给出解决方法）

4 注意事项（可选）

5 相关文档（可选）
```

---

### 5. 常见问题（知识解答类）

**写作目标**：解答用户不理解的概念或不知道的操作过程（功能点维度）等

**标题建议**：应用/单据+如何……（例：采购管理中，如何实现"首页卡片实时获取采购入库单最新单据的入库数量并展示"？）

**内容结构**：

```
1 问题描述（基于问题的标题，补充描述业务背景、功能实现效果等内容）

2 解决方法

3 适用版本（可选，需写明版本限制或适用要求，如果不清楚/不确定具体适用版本，可先不写）

4 注意事项（可选，强调方案实施过程中的关键细节、可能遇到的问题及其解决方案等）

5 相关文档（可选）
```

## 专题与学习路径规范

### 专题介绍

- **对该应用的介绍**：该应用的价值、核心亮点等（60-100字）
- **对该专题的内容介绍**：包含的主要内容，提炼出专题的亮点、学习收益等（60-100字）
- **专题封面**：封面文字不宜过多，突出专题名称，注意图片、字体版权

### 学习路径

**路径分类方式**：
1. 按使用者水平，由浅入深：如入门-进阶-高级/了解-上手-使用-实践
2. 按业务流程顺序：如基础设置-凭证处理-账表查询-期末处理

**路径简介**：突出该路径的核心内容或学习收益（15字以内）

**知识名称**：简洁、明确、易理解，同一路径下的知识名称需注意一致性
