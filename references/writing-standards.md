# 金蝶产品文档中文写作规范

## 整体要求

| 要求 | 说明 |
|------|------|
| 准确性 | 没有错误，并且均以事实为依据 |
| 清晰度 | 不能模棱两可或模糊不清；用户可以快速地理解所要表达的信息 |
| 完整性 | 包括模板要求的所有必要信息 |
| 简洁性 | 减少或避免冗余文字 |
| 具体性 | 包含合适的例子、背景信息和上下文、图示、类比等，能够解决用户当下具体的问题 |
| 一致性 | 内容的表现形式需与本规范保持一致 |
| 内容组织 | 内容各部分的组织结构清晰，易于用户理解 |
| 易检索性 | 用户可以简单快速地找到某个条目 |
| 写作风格 | 恰如其分地遵循写作规范和遣词造句 |
| 及时性 | 随着产品的更新和迭代，及时更新文档内容 |
| 合规性 | 需确保文档内容、格式和披露信息符合行业规范、法律法规及内部政策 |

---

## 一、标题规范

### 1.1 样式要求

标题一般分为五级：

| 标题级别 | 含义 | 编号样式 |
|---------|------|---------|
| 一级标题 | 文章主要部分的大标题 | 1 XXXXX |
| 二级标题 | 一级标题下面二级的小标题 | 1.1 XXXXX |
| 三级标题 | 二级标题下面某一方面的小标题 | 1.1.1 XXXXX |
| 四级标题 | 三级标题下面某一方面的小标题 | 1.1.1.1 XXXXX |
| 五级标题 | 四级标题下面某一方面的小标题 | 1.1.1.1.1 XXXXX |

### 1.2 内容要求

- **标题形式**：采用简单词汇、名词词组、短语或简单句形式编写
  - 简单词汇：如"概述"、"总账"
  - 名词词组：如"系统路径"、"产品概述"
  - 短语：
    - 主题词＋动词：如"按钮说明"、"凭证处理"
    - 动词＋主题词：如"配置用户数据"、"校验分录行"
    - 定语＋主题词：如"合同的变更"、"账表的取数逻辑"
    - 介词＋定语＋主题词：如"对业务流程的监控"
  - 简单句：如"本应用与其他应用的关联"

- **标题需概括反映章节的中心内容**

- **同一级标题建议用一种结构表达**

- **标题不能跨级出现**
  - 【错误示例】缺少二级标题：
    ```
    1 XXXXX
    1.1.1 XXXXX
    ```

- **高级标题下不应只有一个次级标题**
  - 【错误示例】二级标题只包含一个三级标题，可省略：
    ```
    1.1 XXXXX
    1.1.1 XXXXX
    1.2 XXXXX
    ```

- **下级标题不重复上一级标题的名字**
  - 【错误示例】
    ```
    1 概述
    1.1 概述
    ```

- **末级标题下应当有正文内容**

- **标题不解释缩略语**
  - 【错误示例】`1.1 BOTP（Business Object Transform Platform）`
  - 【修订后】正文解释：`BOTP是Business Object Transform Platform的缩写。`

## 二、句子规范

### 2.1 基本要求

1. **避免出现错别字**

2. **不使用冷僻、自创或者文言文的词语**，而要使用现代汉语的常用表达方式

3. **使用代词时，需明确指代的内容**，保证只有一个含义
   - 【错误示例】"从管理系统可以查看业务流状态和受其直接影响的员工任务进度。"
   - 【分析】句中"其"指代不明，不清楚到底指代管理系统还是业务流状态。
   - 【修订后】"从管理系统可以查看业务流状态和受业务流状态直接影响的员工任务进度。"

4. **动宾搭配应合理**
   - 【错误示例】"人人差旅提供中大型集团企业员工随时随地发起出差申请、商旅预订、差旅报销以及差标的管控，实现端到端一站式差旅费用报销。"
   - 【分析】"提供"后搭配的是"员工出差申请、……差标管控"，动宾搭配不当。
   - 【修订后】"人人差旅面向中大型集团企业员工，支持随时随地发起出差申请、商旅预订、差旅报销以及差旅标准管控。"

5. **句子应完整、表达清晰准确、避免歧义**
   - 【错误示例】"管理员需要配置管理界面。"
   - 【分析】可以理解为"配置管理的界面"，也可以理解为"配置和管理界面"。
   - 【修订后】"管理员需要配置管理的界面。"

6. **逻辑需严谨**
   - 【错误示例】"业务管理员首页可以筛选不同共享中心、不同用户组等数据，包含当日、当月任务分配及完成情况展示、待办任务状态展示、处理中任务时段分布图、任务处理时效及处理量排名展示四个部分。"
   - 【分析】错误1：首页筛选数据，不符合逻辑；错误2：由"及"和顿号分割开6个部分，但阐述共"四个部分"，前后矛盾。

7. **相同含义的内容使用一致的表述**
   - 【错误示例】"预置进入共享的业务单据，及设置单据进入共享中心后的流转。"
   - 【修订后】"预置进入共享中心的业务单据，及设置单据进入共享中心后的流转。"

### 2.2 长句限制

- **避免使用长句**
- 一个句子建议**不超过100字**（正文3行）
- 不包含任何标点符号的单个句子，或者以逗号分隔的句子构件，长度尽量**保持在20个字以内**

【错误示例】一段话162个字，实际只有一句话：
> "固定资产从卡片新增到过程管理到资产处置的全生命周期实物管理及财务管理，实物管理中新增卡片包括采购转固定资产、在建工程转固定资产、直接新增卡片、其他组织调入或租入的资产，资产使用过程中员工申请领用、移交、盘点和退库，财务管理中常见的资产的原值、折旧方法、资产类别等的变更，资产的拆分、减值和折旧调整，到最后资产的报废处置清理。"

【修订后】拆分为4个句子，采用总-分结构：
> 固定资产支持从卡片新增、过程管理到资产处置的全生命周期实物管理和财务管理。在实物管理模块中，新增卡片支持以下场景：直接新增、采购转固定资产、在建工程转固定资产、其他组织调入或租入资产。此外，实物管理还支持资产使用过程中的资产领用、移交、盘点和退库操作。财务管理模块支持：资产原值计算方法、折旧方法、资产类别变更、资产的拆分、减值和折旧调整、资产的报废处置和清理。

### 2.3 语态与人称

- **尽量使用肯定句表达**，避免使用双重否定句
- **尽量使用主动语态**
  - 【错误示例】"假如此软件尚未被你安装，则需要报备。"
  - 【修订后】"假如你尚未安装此软件，则需要报备。"
- **使用正式的语言风格，建议使用第二人称（您），直接与用户对话**

## 三、减少冗余文字

### 3.1 避免重复性的表述

- **避免重复解释术语和核心词汇**
  - 应确保术语和核心词汇在编写过程中得到统一处理与呈现，并在手册中通过引用的方式加以利用

- **避免重复介绍同一功能或同一操作方法**

- **减少句子中同一词汇的重复使用**

- **避免同义词或近义词的重复**
  - 如"失败和挫折"、"缺点和不足"等短语中的词汇在意义上相近，可精简

- **避免前缀或后缀的重复**
  - 如"再次重复"、"重新新建"中的"再次"和"重新"与"重复"和"新建"的意义有所重叠

### 3.2 避免过度解释

- **避免描写过于技术性的细节或复杂的功能实现逻辑**
  - 超出用户理解范围或需要的技术细节不需要向用户展开介绍

- **避免过度描述通用性信息**
  - 若产品端展示一目了然的字段或按钮，手册中无需再描述
  - 通用字段、按钮如"新增"、"删除"、"启用"、"禁用"、"提交"、"关闭"等，若无特殊含义，则无需解释
  - 【错误示例】"新增：新增单据"、"提交：提交单据"、"删除：删除单据"——应省略不写
  - 【正确示例】只描述核心的或复杂、需特别说明的字段、按钮，聚焦业务应用场景、关键使用要点及注意事项

- **避免描写不必要的背景信息**
  - 如基础常识、标准法条、国家政策等

### 3.3 避免过时的信息

- 避免出现已经过时或不再适用的信息，以防用户执行错误的操作

### 3.4 避免营销类文字

- 明确区分产品手册和营销材料，确保产品手册不包含营销类描述或推销性语言
- **避免使用绝对化词汇**：如"最佳"、"最先进"、"无与伦比"等
- **避免使用促转化词汇**：如"立即购买"、"抓紧订阅"等
- **避免比较性词汇**：如"更先进"、"更高效"、"更实惠"等

### 3.5 避免过度修饰

- **减少或删除不必要的形容词和副词**
  - 【错误示例】"非常特别"、"十分巨大"中的"非常"和"十分"
  - 【错误示例】"实际上"、"基本上"、"确实"、"真的"
  - 【错误示例】"相应的"、"对应的"等限定性修饰词

### 3.6 避免习惯性冗余文字

- **避免不必要的助词或连词**
  - 像"了"、"的"等助词，或者"而且"、"并且"等连词应避免过度使用

- **避免习惯性短语中的冗余部分**
  - 如"在……的情况下"、"对于……而言"、"对……进行"等，应换一种更精简的方式表达

## 四、段落规范

段落是主题信息块的基本构成单元之一，由多个句子构成：

1. **一个段落只能有一个主题，或一个中心句子**

2. **段落的中心句子放在段首**，对全段内容进行概述。后面陈述的句子作为说明、展开、论据陈述等为核心句服务

3. **一个段落的长度不能超过7行**，最佳段落长度小于等于4行

4. **段落的句子语气要使用陈述和肯定语气**，避免使用感叹语气

## 五、项目列表规范

### 5.1 样式要求

- 在正文中，**存在并列信息块时使用**，单独的一项内容不使用项目列表样式
- 项目列表**不受标题层级限制**，各标题层级下均可使用

**列表类型**：
- **有序列表**：用于并列信息块间存在一定的先后顺序、优先级等逻辑关系，或需要强调数量的场景
- **无序列表**：用于并列信息块无特定顺序的场景

**多级列表样式**：

| 层级 | 有序项目列表样式 | 无序项目列表样式 | 缩进要求 |
|------|-----------------|-----------------|---------|
| 一级项目列表 | ⑴ XXXXX | XXXXX | 社区：无缩进 / Word：2厘米 |
| 二级项目列表 | ① XXXXX | XXXXX | 社区：无缩进 / Word：2.75厘米 |
| 三级项目列表 | XXXXX | XXXXX | 社区：无缩进 / Word：3.5厘米 |

- 有序列表和无序列表可混合使用，且项目列表层级不相互影响
- 项目列表下可嵌套子列表

### 5.2 内容要求

使用项目列表进行编排的内容，其描述应该符合：
- 简单词汇、短语描述
- 简单句描述

## 六、表格规范

### 6.1 样式要求

- 表格一行中的单元格要求**不跨页**
- 对于多行合并单元格的情况，应将段落属性设置成不跨页
- 每个表格下要**空一行**，空行的样式为正文
- 在遇换页时，若表格后的空行为页面第一行，需删除空行
- **表格不增加表名**

**Word格式**：
- 表头：白底黑色边框，文字左对齐且加粗
- 表格内容：白底黑色边框，文字左对齐且不加粗

**社区格式**：
- 表格使用默认样式（白底灰色边框）
- 表头文字左对齐且加粗
- 表格内容文字左对齐且不加粗

### 6.2 内容要求

- 同一文档中相同类型表格的**表头内容需要保持一致**
  - 例如对字段进行说明的表格，不要出现"字段说明"、"字段解释"、"字段含义"等不同的表头内容

- 表格中的内容应该尽量简练，文字表述风格保持一致
- 避免长篇大段的说明，建议单元格中的内容所占行数**不要超过6行**

- **不出现空白的单元格**，建议无内容单元填写"-"

- 单元格内容适当重复
  - 如果多个单元格中的内容相同，建议将内容复制或者采用多个单元格合并的方式，**不要使用"同上"**

## 七、图形规范

### 7.1 使用原则

- **最小化原则**：以降低产品国际化的翻译成本，同时避免文档中过度使用图形带来的竞对风险
- **仅在必要情况使用图形**：文字描述不足以清晰阐述信息时（用文字替代描述超过200字），借助图形补充说明
  - 复杂业务流程或产品整体结构
  - 需对外展示产品的界面设计
  - 抽象的概念或技术，文字难以准确描述
- **仅使用允许公开的图形**：可对外发布的图形版本，内容无敏感信息

### 7.2 样式要求

- 使用**静态图片格式**（如PNG），不建议使用动态格式（GIF）
- 产品文档所使用的图形，**等比例放大或缩小**，确保画质清晰
- 手绘的图形，**底色统一为白色**，图内字体采用"微软雅黑"
- 同一文档的图形**色彩风格保持一致**，产品截图的主题色选择"商务蓝"
- **必要代码不通过截图呈现**，建议使用文档编辑器的代码块

### 7.3 内容要求

- 产品界面截取的图形，**仅体现文档所要描述的局部范围**
  - 不出现产品的菜单栏、应用栏、浏览器
  - 允许出现导航栏、过滤面板、操作区域

- 文档所要描述的图形元素，应在图形的**中心位置**，或使用**方框给予突出表示**

- **图形中不额外输入文字**，对图形的描述性语言放到图外，图内使用数字编号替代

- 图外的描述性语言**必须具体，与图形精准对应**
  - 例如：第1步，导航栏/过滤面板/操作区域/弹窗+上/中/下/左/右方的X功能/按钮+有XX效果

- 图形与图外描述说明，在文档中为**前后相邻关系**

- 图形中出现的数据，**贴近真实业务**，不使用不专业的测试数据
  - 【正确示例】"XX集团"、"XX有限公司"
  - 【错误示例】"测试的哈哈哈"、"test"

- 同一文档对图形标题的使用保持一致，均使用或均不使用，使用时置于图形下方居中

- **图形内不出现金蝶密级标识**

## 八、警告、注意和说明

### 8.1 样式要求

- 中文的文档在警告、注意、说明**下面空一行**，空行的样式为正文样式
- 中文的文档在警告、注意、说明**文字为斜体**，根据分类使用不同颜色
- 警告、注意、说明中如需罗列多个条，请使用项目符号样式

| 分类 | 图标 | 文字颜色 | 使用原则 |
|------|------|---------|---------|
| 警告 | 社区：无图标<br>Word：⚠️ | 红色<br>RGB(255,0,0) | 用于提示用户禁止操作。若用户忽略警告标志，可能会因误操作发生重大事故（如数据丢失、系统重大报错等） |
| 注意 | 社区：无图标<br>Word：⚠️ | 橙色<br>RGB(237,125,49) | 用于提醒用户操作时需注意的事项。若用户忽略注意标志，可能会因为误操作导致无法成功操作或产生一定程度的不良后果 |
| 说明 | 社区：无图标<br>Word：ℹ️ | 黑色<br>RGB(0,0,0) | 用于给用户进行补充说明和提示。若用户忽略说明标志，可能会对文档内容存在一定程度的疑惑或误解 |

### 8.2 内容要求

- 警告、注意、说明中**不能包含表格和图片**
- 说明、注意类的内容不能过长，建议**不要超过4行**

### 8.3 示例

**警告示例**：
> *警告：修改此配置会影响系统安全性。*

**注意示例**：
> *注意：*
> - 成组单据成本检查必须通过，如出现异常，会影响对账结果。
> - 检查零成本数据时，需由用户识别这类单据的零成本是否合法，如不合法，需进行修正。

**说明示例**：
> *说明：检查任务可通过新增、修改权限进行控制，只赋权给财务负责人或IT负责人或系统管理员。*

## 九、标点符号规范

### 9.1 句号

- 句号的形式为"。"，常用于陈述句末尾的停顿
- 在产品资料中常用于简单句和复合句的结尾，表示句子意思已完整
- **在正文描述下面紧接图或表时**，句子说明已完整，**以句号结束，不要使用冒号**
  - 【常见错误】"用户进入管理界面，如图3-2所示："
  - 【修改后】"用户进入管理界面，如图3-2所示。"

### 9.2 逗号

- 逗号的形式为"，"，表示句子内部的一般性停顿
- 一个较长的句子，需要在适当的地方使用逗号加以停顿
- **不能滥用**，经常出现的错误是"一'逗'到底"，整个段落除了段落结尾外，全部停顿都使用逗号

### 9.3 顿号

- 顿号的形式为"、"
- **句子内部并列词语之间的停顿**，用顿号

### 9.4 分号

- 分号的形式为"；"
- **复句内部并列分句之间的停顿**，用分号
- 分行列举的各项之间，可以用分号
- 分项说明时，如分几大项，大项下面又分几个小项；小项之间用分号，大项之间用句号

### 9.5 冒号

- 冒号的形式为"："
- 在手册中常用在需要解释的词语后边，表示引出解释
- 常用来引出Item list和Item Step样式
- **建议不要连续使用冒号**
  - 【常见错误】"计费策略：表示计费：1，表示不计费：2。"
  - 【建议修改】"计费策略：1表示计费，2表示不计费。"

### 9.6 引号

- 引号的形式为双引号""和单引号''
- **引号里面还要用引号时，外面一层用双引号，里面一层用单引号**
- 产品手册的正文中若涉及**特定参数名称、字段名、界面字段、引用手册的节选**等，建议使用引号

### 9.7 括号

括号常用的形式是圆括号"（）"，此外还有方括号"【】"和尖括号"<>"

**特定用法**：
- **方括号"【】"**：表示窗口名、菜单名等
  - 如"弹出【新建用户】窗口"、"【文件/新建/文件夹】多级菜单"
- **尖括号"<>"**：表示按钮名、键名
  - 如"单击<确定>按钮"、"<Tab>表示制表键"

### 9.8 连接号

| 符号名称 | 符号 | 举例 | 使用场景 |
|---------|------|------|---------|
| 一字线 | — | 压力—温度曲线、化学—物理反应 | 常用于汉语名词之间 |
| 半字线 | – | 铀–235、惠普–5000 | 常用于汉字和数字（字母）之间 |
| 短划线 | - | E-Mail、CD-ROM、RJ-45插头、025-12345678 | 外来词汇、电话号码 |
| 浪纹 | ~ | 1~3 | 表示数量的范围 |

### 9.9 书名号

- 书名号的形式为双书名号"《》"
- **书名、篇名、报纸名、刊物名**等，用书名号标示
- **英文手册名称不需要书名号标示**
- 书名号中的书名、片名等全称，**不能使用简称**

### 9.10 破折号

- 破折号的形式为"——"，占两个字的位置
- 技术文献中破折号常用于**表示注释和说明部分之前**
- 也常用于连接地名等，如：北京——天津

### 9.11 省略号

- 省略号的形式为"……"，六个小圆点，占两个字的位置
- **引文的省略**，用省略号标明
- **列举的省略**，用省略号标明

### 9.12 标点符号使用说明

- 句号、问号、叹号、逗号、顿号、分号和冒号**不出现在一行之首**
- 引号、括号、书名号的前一半**不出现在一行之末**，后一半**不出现在一行之首**
- 破折号和省略号都占两个字的位置，**中间不能断开**
